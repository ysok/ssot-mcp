Import GitHub organization

Lists public repos via the GitHub API, then processes each in the background: first every mirror is cloned or synced and keyword-indexed; only after that pass finishes does the server queue semantic (embedding) indexing repo-by-repo so large repos cannot block the rest. For large orgs, set a GitHub API token or GITHUB_TOKEN / GITHUB_PERSONAL_ACCESS_TOKEN in the environment to avoid rate limits.

Each git operation uses a timeout (clone default 10 minutes). For very large repositories, set SSOT_GIT_CLONE_TIMEOUT / SSOT_GIT_SYNC_TIMEOUT on the server (seconds, up to 86400). You will be redirected to a live progress page and can browse Repositories while the import runs.
