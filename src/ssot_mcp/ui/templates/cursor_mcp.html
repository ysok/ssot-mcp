Connect Cursor to this MCP server

This admin UI and the MCP server are separate processes (or the container runs both). Cursor talks to the MCP endpoint only—not to this browser UI.

MCP URL for this deployment: {{ mcp_url }} — from FASTMCP_HOST / FASTMCP_PORT (defaults 0.0.0.0 / {{ fastmcp_port }}; localhost shown when the server listens on all interfaces).

1. Run the MCP server (HTTP)

From a terminal (same machine as Cursor, or reachable on your network):

export SSOT_DATA_DIR=./data
export FASTMCP_HOST=127.0.0.1
export FASTMCP_PORT={{ fastmcp_port }}
python -m ssot_mcp.mcp.server

Container / combined runtime: ensure port {{ fastmcp_port }} is published (e.g. -p {{ fastmcp_port }}:{{ fastmcp_port }}).

2. Add the server in Cursor

Use either the project file .cursor/mcp.json or the user file ~/.cursor/mcp.json. In Cursor Settings you can also use Tools & MCP → add server.

For a remote HTTP server, Cursor expects streamable HTTP transport (matches this project’s default). Example:

{
  "mcpServers": {
    "ssot-mcp": {
      "type": "streamableHttp",
      "url": "{{ mcp_url }}"
    }
  }
}

If your Cursor build only shows a URL field for HTTP MCP, use the same URL: {{ mcp_url }}.

After editing JSON, fully restart Cursor so the server list reloads.

Alternative: stdio (Cursor starts the server)

Use this when you want Cursor to spawn the process instead of connecting to a URL. Point command at the same Python where ssot-mcp is installed, and set SSOT_DATA_DIR to your data root.

{
  "mcpServers": {
    "ssot-mcp": {
      "command": "python",
      "args": ["-m", "ssot_mcp.mcp.server"],
      "env": {
        "SSOT_DATA_DIR": "/path/to/your/data",
        "SSOT_MCP_TRANSPORT": "stdio"
      }
    }
  }
}

On Windows, use your venv’s python.exe path. Remove FASTMCP_HOST / FASTMCP_PORT for stdio—they apply to HTTP mode only.

Same environment as this UI

MCP tools use the same SSOT_DATA_DIR as this admin UI (same SQLite registry and mirrors). Semantic search uses on-device embeddings (pip install 'ssot-mcp[semantic]' — LanceDB + fastembed; no cloud API keys). Use the same SSOT_DATA_DIR in stdio env if Cursor starts the server.

Check that it works

  • In Cursor chat, ask what MCP tools are available—you should see list_repositories, search_code, etc.
  • Ensure SSOT_DATA_DIR is the same for the MCP process as for this UI (same registry and mirrors).
